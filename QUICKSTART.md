# 快速参考指南

## 启动应用

### Windows
```bash
# 方式 1: 双击启动脚本
start.bat

# 方式 2: 手动启动
# 终端 1
cd backend
uvicorn main:app --reload --port 8000

# 终端 2
cd frontend
npm run dev
```

### Linux / macOS
```bash
./start.sh
```

## 功能快速导航

### 🌍 分布识别分析

**目的**: 查看物种分布区域

**步骤**:
1. 点击第一个 Tab "🌍 分布识别分析"
2. 从下拉菜单选择物种
3. 在地图上会显示该物种的所有位置标记
4. 查看"已记录点位"统计数据

**快捷键**: 
- 鼠标滚轮缩放地图
- 按住拖动移动地图

**支持的物种**: 豹纹翼甲鲶、大鳄龟、鳄雀鳝、非洲大蜗牛、福寿螺、红耳彩龟、美洲牛蛙、齐氏罗非鱼

---

### 🤖 智能知识问答

**目的**: 了解物种的危害、防治方法等信息

**步骤**:
1. 点击第二个 Tab "🤖 智能知识问答"
2. 从右侧物种面板选择某个物种，或直接输入问题
3. 在底部输入框输入你的问题
4. 点击"发送"或按 Enter
5. 等待 AI 回答

**预设问题**:
- "介绍一下 [物种]"
- "[物种] 的危害是什么？"
- "如何防治 [物种]？"
- "[物种] 属于什么分类？"
- "[物种] 的原产地在哪？"

**功能**:
- 💡 快速建议按钮 - 点击建议的问题快速提问
- 🔄 清空对话 - 开始新的对话

**示例对话**:
```
你: 福寿螺有什么危害？
助手: 福寿螺是入侵物种，对农业造成严重危害...
```

---

### 📝 数据上报与更新

**目的**: 报告发现的物种位置

**步骤**:
1. 点击第三个 Tab "📝 数据上报与更新"
2. 在表单中填写以下信息：
   - **物种名称** (必填) - 从下拉菜单选择
   - **位置名称** (必填) - 例如"太湖"、"长江主干道"
   - **纬度** (必填) - -90 到 90 之间
   - **经度** (必填) - -180 到 180 之间
   - **发现日期** (可选) - 默认为今天

3. 点击"💾 保存记录"
4. 看到"✅ 记录保存成功"的提示
5. 下方表格会显示新增的记录

**表单验证**:
- 所有必填项都需要填写
- 坐标范围会自动验证
- 保存成功后表单自动清空

**获取坐标的方式**:
1. 打开 Google Maps 或高德地图
2. 在地图上找到位置
3. 右键点击，复制纬度和经度
4. 粘贴到表单中

**示例提交**:
```
物种: 福寿螺
位置: 杭州西湖
纬度: 30.2741
经度: 120.1551
日期: 2024-04-01
```

## API 使用示例

### 获取物种列表
```bash
curl http://localhost:8000/api/species
```

### 获取物种分布
```bash
curl http://localhost:8000/api/locations/福寿螺
```

### 提问
```bash
curl -X POST http://localhost:8000/api/qa \
  -H "Content-Type: application/json" \
  -d '{"question": "福寿螺怎么防治？"}'
```

### 上报数据
```bash
curl -X POST http://localhost:8000/api/record/location \
  -H "Content-Type: application/json" \
  -d '{
    "species": "福寿螺",
    "latitude": 30.27,
    "longitude": 120.15,
    "location_name": "杭州西湖",
    "date": "2024-04-01"
  }'
```

## 常见问题 (FAQ)

**Q: 为什么地图不显示？**
A: 检查后端是否在运行，物种列表是否加载成功

**Q: 为什么 AI 回答不了？**
A: 检查 API Key 是否配置，或检查网络连接

**Q: 能否导出所有数据？**
A: 整个 locations_record.csv 文件就是数据库，可以直接导出

**Q: 如何删除上报的记录？**
A: 联系管理员或手动编辑 locations_record.csv

## 性能建议

- 物种位置超过 1000 时，页面可能会变慢
- 建议在 7 次错误回答后清空对话
- 大型数据导入时，建议在非高峰时段进行
